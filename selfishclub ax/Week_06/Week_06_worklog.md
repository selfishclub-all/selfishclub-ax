### 삽질과 배움

**전화번호 포맷 불일치로 DB 중복 발생**
폼 placeholder에 `010-0000-0000`으로 안내하고 있었는데, 입력값을 정규화 없이 그대로 DB에 저장하고 있었다. 기존 데이터는 대시 없이 저장되어 있어서 같은 사람이 다른 번호로 인식돼 membership이 중복 생성되고, 신규/재방문 판별도 틀어졌다.
→ **배움: 전화번호처럼 다양한 형태로 입력될 수 있는 필드는 반드시 저장 전에 정규화해야 한다. DB에서 식별자로 쓰이는 값은 특히 중요.**

**Webflow Embed에서 코드 줄바꿈으로 HTML 깨짐**
Webflow Code Embed에 여러 줄 HTML을 넣었더니 줄바꿈 위치에서 태그가 잘려서 `<span>` 같은 텍스트가 그대로 표시됐다. 한 줄로 압축해서 넣으니 해결.
→ **배움: Webflow Embed에 코드를 넣을 때는 한 줄로 압축해서 넣어야 줄바꿈으로 인한 깨짐을 방지할 수 있다.**

### 삽질과 배움

**Supabase select()의 기본 1,000행 제한을 몰라서 버그 발생**
누적신청자 수를 세는 로직에서 `.select("ID").eq("iid", item.iid)`로 전체 행을 가져와 `.length`로 세고 있었는데, Supabase는 기본적으로 최대 1,000행만 반환한다. 1,000명이 넘자 카운트가 멈췄다. `count: "exact"` + `head: true` 옵션으로 DB 레벨 카운트를 사용하도록 변경.
→ **배움: Supabase에서 행 수를 세야 할 때는 전체 데이터를 가져와서 .length로 세지 말고, count: "exact" 옵션을 사용해야 한다. 성능도 좋고 1,000행 제한에도 걸리지 않는다.**

**비주얼 컴패니언(브레인스토밍 도구) 서버가 30분 후 자동 종료**
스폰지클럽 디자인 방향을 잡을 때 비주얼 컴패니언으로 컬러 팔레트/레이아웃을 브라우저에서 확인하며 작업했는데, 중간에 서버가 자동 종료되어 새 포트로 다시 켜야 했다. 30분 inactivity timeout이 있다는 걸 모르고 있었다.
→ **배움: 비주얼 컴패니언은 빠른 의사결정에 유용하지만, 장시간 세션에서는 서버 재시작이 필요할 수 있다. 핵심 결정만 빠르게 하고 나머지는 로컬 dev 서버에서 확인하는 게 효율적이다.**
