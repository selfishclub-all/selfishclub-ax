### 삽질과 배움

**JSX 태그 불일치 에러로 페이지 빌드 실패**
타이틀 수정 시 `<h2>` 태그를 `</p>`로 닫는 실수를 해서 빌드가 깨졌다. 기존 `<h2>` + `<p>` 2줄 구조를 `<h2>` 1줄로 합치면서 닫는 태그를 미처 수정하지 않은 것. 에러 메시지가 "Expected '</', got 'jsx text'"로 나와서 바로 원인을 찾을 수 있었다.
→ **배움: 태그 구조를 변경할 때 여는 태그와 닫는 태그를 반드시 함께 확인해야 한다.**

**replace_all로 opacity를 일괄 변경하니 의도치 않은 곳까지 영향**
가독성 개선을 위해 `opacity-40` → `opacity-60` 등을 일괄 치환했는데, 배경 요소의 투명도까지 함께 올라가서 일부 디자인이 의도와 달라졌다. 다행히 큰 문제는 없었지만, 선택적으로 적용했어야 했다.
→ **배움: replace_all은 편리하지만, UI 속성값을 일괄 변경할 때는 맥락별로 영향도를 먼저 확인해야 한다.**

### 삽질과 배움

**STOCK_LIMIT=0이 전체 결제를 차단하고 있었던 문제**
스폰지클럽 매진 처리를 위해 `STOCK_LIMIT = 0`으로 설정했던 코드가 결제 API 전체에 하드코딩되어 있어서, 새로 등록한 프로그램도 모두 결제가 차단되고 있었다. `sold >= 0`은 항상 true이므로 모든 결제가 자동 환불 처리됨.
→ **배움: 하드코딩된 제한값은 특정 상품용이라도 공용 API에 넣으면 전체에 영향을 준다. 상품별 설정은 DB에서 관리해야 한다.**

**행사 당일인데 "종료됨"으로 표시되는 문제**
`new Date("2026-04-30")`은 UTC 자정(= KST 오전 9시)으로 파싱된다. 한국 시간 오전 9시 이후에는 이미 지난 날짜로 판정되어 결제 버튼이 숨겨졌다.
→ **배움: 날짜만 있는 문자열(`YYYY-MM-DD`)은 UTC로 파싱되므로, KST 기준 판정이 필요하면 시차(+9시간)를 반드시 고려해야 한다.**
