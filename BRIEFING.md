## 아키텍처 원칙

**프론트엔드에서 Supabase 직접 호출 금지.**
반드시 Next.js API Routes를 중간 레이어로 두고 서버 사이드에서만 Supabase에 접근할 것.

```
[사용자] → [Next.js 프론트엔드]
                ↓
    BetterAuth 미들웨어 (인증 검증)
                ↓
    [Next.js API Routes]
                ↓
    [Supabase PostgreSQL]
```

## Supabase 기존 테이블 (건드리지 말 것)

기존 테이블은 절대 삭제/수정하지 말 것. 구조 변경 필요 시 반드시 확인 후 진행.

| 테이블 | 역할 |
|---|---|
| member | 멤버십 이전 신청자 (구버전) |
| membership | 현재 멤버십 가입자 |
| item | 프로그램 정보 |
| event | 이벤트 정보 |
| purchase | 결제/신청 내역 |
| refund_request | 환불 요청 |
| reviews | 후기 |
| newsletter_DB | 뉴스레터 구독자 |
| newsletter_bot_config | 뉴스레터 봇 설정 |
| test_membership / test_purchase | 테스트용 |

BetterAuth가 자동 생성하는 테이블: `user`, `session`, `account`

## Auth 구조

- BetterAuth + 카카오 OAuth
- 비밀번호 기능 OFF
- 유저 데이터는 우리 Supabase DB에 직접 저장

### 기존 회원 매핑 플로우

카카오로 첫 로그인 시 기존 회원과 연결해야 함:

```
카카오 로그인
    ↓
전화번호로 membership 테이블 매핑 시도
    ↓
성공 → 기존 내역 연결
실패 → 이메일 입력 요청 → 이메일로 매핑 시도
    ↓
성공 → 기존 내역 연결
실패 → 신규 유저로 처리 → 카카오 채널 문의 안내
```

**주의:** 카카오 전화번호 수집은 비즈앱 인증 + Developer Console 동의 항목 활성화 필요.

## 셀피쉬클럽 브랜드 & 톤앤매너

Claude API로 콘텐츠를 생성할 때 반드시 참고할 것.

### 핵심 표현 (보존할 것)
- 냅-다 (bias-to-action)
- 쌩-날 (날것의 실전 경험)
- 이기적 공유 (핵심 철학)
- 디지털 잡부 (멀티 스킬 정체성)
- 우당탕탕 (혼란스럽지만 해내는)

### 톤 규칙
- 반말/친근체, 고에너지, 실전 중심, 동료 대 동료
- 절대 쓰지 말 것: "AI로 업무 효율을 높이세요", "함께 성장해요", 습니다체, 에이전시/SaaS 말투

### 조직 구조 (카피 작성 시 참고)
```
Selfish Club (유니버스)
├── 이기적멤버십 — 공개 멤버십 (누구나 가입 가능)
├── AAA — 클로즈드 AI 네이티브 커뮤니티 (코호트 입장)
├── 셀피쉬월드 — 연간 대형 포럼
├── 천스체 — 스포츠 네트워킹 행사
├── 이기적공유회 — 무료 오픈 세미나
└── 셀피쉬크루 — 전체 운영진
```

**이기적멤버십과 AAA는 위계 없는 병렬 구조** — 파이프라인이 아님.
