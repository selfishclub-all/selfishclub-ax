## 언제 이 메모리 쓰는가
자동화 재개 기획/설계 세션 시작 시 **반드시 먼저 읽기**. 4/21 AAA CRM 이 수동 플레이북 = 자동화 재설계의 출발점(SSOT).

## 🔁 수동 CRM 전체 흐름 (4/21 AAA 실행 기록)

### Phase 0 — 준비 단계 (오픈일 전날)
1. 공유회 정보 파악 (item 테이블): iid, 제목, 일정, 오픈일
2. 상세페이지 크롤 → 핵심 소구점/타겟/커리큘럼/혜택 추출
3. 사전 설문 CSV 확보 (로컬 파일, 경로는 매번 사용자가 지정)
4. 지난 공유회 카피풀 조회 (겹침 방지)

### Phase 1 — 모수 분석 (오픈일 오전)
1. 멤버십 모수: `u_membership_tf=true AND u_phone IS NOT NULL`
2. 전화번호 정규화 + 고유번호 dedup
3. AAA 현재 신청자 조회 (event 테이블, iid_XXXX) — purchase가 아님 주의
4. 공통 제외: AAA 신청자 + 테스트 iid 참여자

### Phase 2 — 세그먼트 배정 (우선순위 체인)
1. 각 세그먼트 정의 iid 그룹 로드 (클로드공유회/워크숍/AI영상 등)
2. `u_participate_iids`로 S5 로열 판정
3. `u_recent_date`로 S6 휴면 판정
4. `u_item_tf`로 S7 첫진입 판정
5. **상위 세그먼트부터 순차 배정**, 이미 배정된 사람은 skip
6. 합계 = afterExclude 검증

### Phase 3 — 데이터 정제
1. 이름 test/A/admin/빈값 제외
2. 전화 10자리 이메일 매칭 복구 (동일 이메일의 다른 row에 정상 11자리 있으면 교체)
3. 전화 01 미시작/10자리 복구불가 → 정제제외

### Phase 4 — XLSX 통합
시트 구성 (13개): 요약 / S1~S8 / AAA신청_설문자 / AAA신청_비설문 / 정제제외 / 전화번호복구

### Phase 5 — 카피 기획
1. 세그먼트 × T1/T2 배분 결정 (기본 T1+T2 병렬)
2. T1용 `#{프로그램설명}` 작성 (18~45자)
3. T2용 `#{안내문구}` 세그먼트별 작성
4. 사용자 검증 → Slack `#crm-카피` 포스트

### Phase 6 — 발송 (수동)
1. Slack `#crm-알림톡-승인단계`에 카피 + 대상자 CSV 전달
2. 비비안/주희가 SOLAPI에서 배치 발송
3. 발송 완료 시각/대상자 수 Slack 기록
