## AAA 팀 주차별 미션 과제

### 경로
- **Vault 루트**: `~/Projects/AAA/aaa/` (로컬, iCloud 아님)
- **GitHub remote**: `https://github.com/selfishclub-all/aaa.git`
- **과제 파일**: `~/Projects/AAA/aaa/00_주차별미션/Week_XX/Week_XX_에밀리.md`
- **주차 정의**: AAA 모임 주차 번호 (달력 주 ≠ AAA 주차). 현재 진행 중인 Week는 git status에서 modified 상태인 파일로 확인
- **커밋/푸시 장소**: vault 자체가 git repo. `cd ~/Projects/AAA/aaa && git add 00_주차별미션/Week_XX/Week_XX_에밀리.md && git commit && git push`

### 독자
- 1차: AAA 팀원 (클로드코드 사용자, CRM 지식 없음)
- 2차: **외부 공개** (밖에도 게시됨) → 외부 독자 이해 가능하게 작성

## 📝 포맷 규칙 (5주차~ 업데이트)

### 1. 🗺️ 전체 구조도 먼저 (NEW)
**"오늘 이야기할 거는 무엇무엇임"** — 글 맨 처음에 big picture 제시:
- 한두 줄로 "오늘 이야기할 것" 요약
- 1, 2, 3, 4 포인트 먼저 나열 (네비게이션 역할)
- 독자가 뭘 보게 될지 한눈에 잡히게
- 그 다음에 세부로 들어가기

### 2. 🧠 유저/페르소나 관점 필수 (NEW — 5주차 핵심)
**에밀리 = 에밀리 본인이 고객** (셀피쉬클럽 CRM을 본인이 직접 운영)

그래서 "에밀리 OS" 관점으로 녹여야 함:
- **왜 이걸 만들었는가** — 어떤 pain point, 어떤 배경
- **어떻게 쓰는가** — 실제 본인 운영 플로우에 어떻게 녹아드는가
- **이게 나한테 어떤 가치를 주는가** — 시간 단축? 실수 감소? 자동화?

지금까지(~4주차)는 **"Claude Code 어떻게 써"** 중심이었다면, 5주차부터는 **"내가 왜 만들었고, 내 관점에서 이게 어떤 가치를 주는지"** 축으로 이동.

### 3. 🛠️ 삽질 과정 최소 1~2개 (강화)
전체 얘기하느라 시간 부족해도, **최소 1~2개 포인트는 반드시 공유**:
- As-is → To-be 구조
- 왜 삽질했는지 (원인 분석)
- 어떻게 풀었는지 (해결 과정)
- 배운 것

### 4. 📊 결과물 (PR 링크 + 한줄 요약)
- GitHub PR 또는 커밋 해시 링크
- "이번 주에 뭐 했는가" 1~2줄

### 5. 💡 추가 인사이트 (구조적 깨달음)
- 도구 비교, 아키텍처 결정, 트레이드오프 등

## 🗣️ 말투/톤

- **혼자 기록하는 노트 톤**: "~완료", "~확인", "~으로 변경", "~세팅 완료"
- ❌ 반말 X ("~했다", "~됐다")
- ❌ 존댓말 X ("~했습니다")
- ✅ 기록체: "세팅 완료", "확인됨", "이걸로 결정", "여기서 삽질"
- 노션 스타일 들여쓰기 + 볼드 강조
- 스크린샷 자리는 비워두기 (옵시디언에서 직접 추가)
- 외부 공개 의식 → 내부 용어/맥락은 간단히 풀어서
- **일요일에 전체 재정리**: 주중 누적된 내용을 일요일에 한 번에 다시 정리. 항목 하나하나 빠짐없이 기록. "추가됨" 같은 뭉뚱그리기 금지.

## 🔄 일일 누적 작성 워크플로우 (NEW — 4주차부터)

**일요일에 몰아 쓰지 않는다. 매일 세션 끝날 때 누적 업데이트한다.**

### 왜 이렇게 하나
- 세션 컨텍스트는 휘발됨 — 맥북 바꾸거나 새 대화 열면 오늘 한 일 기억 안 남
- 메모리 파일에 적재하면 다음 세션 Claude가 읽고 이어 쓸 수 있음
- 일요일에 몰아 쓰면 주 초반 디테일 다 까먹음

### 트리거 & 동작
1. **"오늘 과제 작성해줘"** / **"옵시디언 과제 작성해줘"** 하면:
   - 해당 주차 MD 파일 경로 확인 (`Week_XX/Week_XX_에밀리.md`)
   - 파일 있으면 → **기존 내용 Read 후 오늘 작업분 누적 업데이트**
   - 파일 없으면 → **포맷 스켈레톤 + 오늘 작업분으로 신규 생성**
   - git log, 진행 중인 todo, 오늘 세션 대화 기반으로 작성

2. **"과제 업데이트해줘"** 하면:
   - 현재 주차 파일 Read
   - 이번 세션에서 새로 한 일 추가
   - 기존 섹션 구조는 유지, 내용만 보강
   - 중복되는 건 통합, 시간 순서 유지

3. **일요일 마무리**:
   - 주중 누적된 내용 전체 리뷰
   - **발표 가이드 섹션 완성** (10~15분 기준)
   - 최종 정리본 완성

### 파일 구조 (누적 작성 시)
주중 누적 단계에서는 **날짜별 섹션으로 쌓음**:

```markdown
# Week 04 — 에밀리

## 📋 오늘 이야기할 것 (작성 중 — 일요일에 최종 정리)
- [ ] 포인트 1
- [ ] 포인트 2
- [ ] 포인트 3
- [ ] 포인트 4

---

## 📅 4/7 (월)
### 결과물
- 커밋: [해시] 한 줄 요약

### 삽질
- 뭐가 안 됐고, 어떻게 풀었는지

### 인사이트
- 배운 것

---

## 📅 4/8 (화)
...

---

## 📅 4/9 (수)
...
```

**일요일 최종 정리 시**: 위 날짜별 섹션을 **포인트 1~4 구조로 재조직** + 발표 가이드 추가.

### 세션 간 핸드오프 포인트
- 파일이 SSOT (Single Source of Truth)
- 새 세션 Claude는 파일 읽고 "어디까지 썼는지" 파악
- 메모리 파일 + 옵시디언 과제 파일 둘 다 보고 컨텍스트 복원
- 맥북 바꿔도 iCloud로 동기화되니까 OK

## 🎤 발표 가이드 섹션 (과제 마지막에 반드시 포함)

과제 본문이 끝나면 **"발표 가이드" 섹션**을 글 맨 끝에 추가한다.
글 정리 + 발표 준비가 동시에 끝나게 하기 위함.

### 포맷 (10~15분 기준)

```markdown
---

## 🎤 발표 가이드 (10~15분)

### 오프닝 (1분)
"오늘 이야기할 건 크게 4가지야:
1. [한 줄]
2. [한 줄]
3. [한 줄]
4. [한 줄]"
→ 전체 구조도/그림 한 장 (다이어그램 또는 목차)

### 본론 (7~10분)
**포인트 1** (2~3분)
- 핵심 메시지 한 줄
- 말로 어떻게 풀지 힌트 3~4개
- 근거/예시/스크린샷 위치

**포인트 2** (2~3분)
- 핵심 메시지 한 줄
- 말로 어떻게 풀지 힌트 3~4개

**포인트 3** (2분)
- 핵심 메시지 한 줄
- 말로 어떻게 풀지 힌트 2~3개

**포인트 4** (1~2분)
- 핵심 메시지 한 줄
- 말로 어떻게 풀지 힌트 2~3개

### 삽질 스토리 (2~3분, 1~2개)
"사실 이번 주 제일 크게 삽질한 건 XX인데..."
- 핵심 한 문장 요약
- 왜 삽질했는지 (원인, 구체적으로)
- 어떻게 풀었는지 (해결 과정)
- 배운 점/구조적 깨달음

(두 번째 삽질 있으면 간단히 1분)

### 유저 관점 클로징 (1분)
"나는 내 CRM을 내가 운영하는 입장이라서, 이게 나한테는..."
- 에밀리 OS에서의 의미
- 1차 버전이 나한테 어떤 가치를 주는가
- 다음 주/다음 단계 한 줄

### Q&A 대비 (1~2분)
- 예상 질문 1: [질문] → [답변 포인트]
- 예상 질문 2: [질문] → [답변 포인트]
- 예상 질문 3: [질문] → [답변 포인트]
- 예상 질문 4: [질문] → [답변 포인트] (여유 있으면)
```

### 발표 가이드 작성 규칙
- **시간 배분 명시** (1분 / 7~10분 / 2~3분 / 1분 / 1~2분)
- **말로 어떻게 풀지 힌트** — 글 본문을 그대로 읽는 게 아니라, 말로 전달할 때의 요약 키워드
- **포인트별 시간 할당** — 어느 포인트에 더 시간 쓸지 미리 정해두기
- **Q&A 예상** — 듣는 사람 관점에서 뭐가 궁금할지 3~4개 미리 준비
- **에밀리 OS 관점**으로 클로징 고정 (5주차~)
- **스크린샷/다이어그램 위치** 본론 각 포인트에 명시

---

**Why:** 에이전트 팀 과제 제출용 + 외부 공개용. 프로젝트 코드에 포함 안 됨.
**How to apply:** "N주차 과제 정리해줘" → git log로 해당 주 커밋 확인 → 위 포맷+체크리스트대로 작성 → 마지막에 발표 가이드 섹션까지 완성
