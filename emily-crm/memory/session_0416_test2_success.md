## 핵심 버그: `Code: 취소 확인` 이중 이스케이프
### 증상
- 크론 정상 실행 (슬랙 미리보기 도착) → 승인 클릭 → **카톡 미도착**
- execution 11255: status=error, webhook 모드 (승인 후 발송 단계에서 실패)
- execution data가 API에서 안 나와서 에러 상세 확인 불가

### 원인
- `Code: 취소 확인` 노드의 jsCode에서 **템플릿 리터럴이 이중 이스케이프**됨
- 정상: `` `ver_${d.t}_${d.iid}` ``
- 실제: `` \`ver_\${d.t}_\${d.iid}\` ``
- **API PUT으로 jsCode 업데이트할 때 발생** — bash heredoc이나 jq 처리 과정에서 백틱/달러사인이 추가 이스케이프됨
- JavaScript 문법 에러 → 노드 실행 실패 → 승인해도 발송 안 됨

### 수정 방법
- **python3으로 JSON payload 생성** (jq 대신) — 템플릿 리터럴이 정확히 보존됨
- PUT → deactivate → sleep 2 → activate

### 비교: 다른 노드는 정상
- `Code: SQL 재생성`: 같은 `` `ver_${d.t}_${d.iid}` `` 사용하지만 이중 이스케이프 없음
- 차이점: SQL 재생성은 백업에서 복원된 원본, 취소 확인은 API로 새로 작성한 코드
