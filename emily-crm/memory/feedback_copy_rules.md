## 혜택 구조 (절대 혼동 금지)

**기본 (신청만 해도):**
- 공유회 다시보기 VOD 제공

**라이브 이벤트 참여 시:**
- 라이브혜택 모음집 (공유회마다 다름, 상세페이지에서 크롤링)

### 표현 규칙
- ✅ "라이브 이벤트 참여 시"
- ❌ "라이브 참여하면" (너무 축약)
- ❌ "라이브 완주 시" (사용 안 함)

→ 알림톡, 오카방, 인스타, 온드미디어 등 모든 채널 카피에 적용

## ⑤ 할인쿠폰 카피 규칙

할인쿠폰 알림톡에는 반드시 아래 안내 포함:
- ✅ "아래 링크에서 쿠폰 자동 적용된 가격으로 신청 가능합니다"
- 사용자가 햇갈리지 않게 쿠폰 적용 방법을 명확히

→ 모달에서 #{쿠폰명} 또는 #{종료일} 카피에 이 안내 포함시킬 것

## 알림톡 카피 레퍼런스 형식

Slack #crm-카피에 보내는 알림톡 카피는 **에밀리가 골라 쓸 수 있게 풀카피 + 혜택 리스트 둘 다 포함**:

```
{프로그램 제목}

{크롤링 기반 카피 — 후킹+내용+체크마크}

🎁 라이브 이벤트 참여 시, {핵심혜택 한줄 요약}!

---

이런 것들을 가져가실 수 있어요!

✅ 신청자 전원
- 공유회 다시보기 VOD

✅ 라이브 이벤트 참여 시
- {혜택1}
- {혜택2}
- {혜택3}
```

→ 에밀리가 모달에서 원하는 부분만 골라 붙여넣기 가능하도록

## 가격 표현 규칙
- ❌ "90% 할인" — 실제 할인이 아님, 쓰지 말 것
- ❌ "99,000원 → 9,900원" — 쓰지 말 것
- ✅ 가격은 카피에 넣지 않음 (온드미디어/오카방/카플친 등)
- ✅ 리마인드/마감에는 "마감 후 최대 할인가로 신청 불가" 식으로 긴급성만

## 🚫 공유자(크루) 실명 사용 금지 (2026-04-22 재강조)

**모든 채널 카피(오카방·알림톡·인스타·카플친·온드미디어 등)에 크루 실명 쓰지 말 것.**

### 규칙
- 크루 이름(젬마/다다/다니/흐민/오웬/띵크/비비안/에밀리 등) 카피 본문에 노출 금지
- 실명 대신 **"한 크루"**, **"저희 크루 중 한 명"**, **"한 멤버"** 등으로 익명화
- 단 예외: **발화자 본인** (에밀리가 본인 쓴 글이면 "저(에밀리)"는 OK) — 본인 외 크루는 실명 금지
- 구체 결과물(터미널에서 영상, 4시간 에이전트 12개, CRM 자동화 등)은 OK. 누구인지만 숨김

### Why
공유회 전 바이럴·홍보 카피에서 실명이 빠지면:
- 호기심 자극 ("누가 했을까?")
- 공유회에서 "그 크루는 OOO였습니다!" 공개 임팩트
- 발표 순서/구성 바뀔 때 카피 수정 부담 없음

### How to apply
- 카피 초안 작성 직후 **"크루 실명 들어있는지"** 전체 스캔
- 발견되면 "한 크루"/"저희 팀 한 명" 등으로 교체
- 알림톡에서 #{이름} 변수는 수신자 이름이라 OK (크루 실명과 구별)

## 카피 톤 규칙 (전 채널 공통)

### 고정 — 톤과 구조
- **자연스러운 구어체**: "~거든요", "~더라구요", "~이에요", "~해보세요"
- **AI 티 절대 금지**: "~했습니다. ~하고요." 식 setup→punchline 금지
- **비개발자 타겟**: 기술 용어(터미널, 깃헙, 크론 등)로 후킹 금지. 결과 중심 (내 서비스 만들기, 실무 활용)
- **구조**: 후킹 → 페인포인트(😰) → 가져갈 것(✅) → 혜택(🎁) → CTA
- **Phase별 각도**: 오픈=호기심/궁금증, 리마인드=FOMO/사회적증거, 마감=긴급성/할인마감
- **대상 프레이밍**: "이런 분들이 들으면 좋아요 👇" (듣고 계세요 X)
- **이모지**: 감정 이모지 적극 활용 (🤔😰🔥👀🚨😱💀)

### 매번 새로 — 공유회마다 다르게
- **후킹 첫 2줄**: 그 공유회만의 핵심 소구점에서 추출. 고정 문구 템플릿 금지.
- **리마인드 숫자**: "벌써 N명 돌파" 같은 건 실제 신청자 수 기반. 매번 다름.
- **페인포인트(😰)**: 그 주제에 맞는 실제 고민. 공유회마다 타겟이 다름.
- **가져갈 것(✅)**: 실제 커리큘럼에서 추출. 복붙 금지, 재해석해서 작성.
- **혜택(🎁)**: 상세페이지에서 크롤링. 공유회마다 라이브혜택 다름.
- **절대 고정 문구 복붙 금지**: 같은 구조여도 텍스트 자체는 매번 새로 작성.
