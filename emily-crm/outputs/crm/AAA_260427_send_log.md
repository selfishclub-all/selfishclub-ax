## 발송 이슈·교훈

### Issue 1 — Supabase 페이지네이션 빠뜨림 (D-1 1차)
- 첫 발송 스크립트(`send-d1-bulk.js`)에서 `?limit=10000` 단일 호출 사용 → Supabase 기본 1000 limit 적용 → first 1000 raw rows만 가져옴 (981 unique)
- 누락 374 raw → unique 347명 발송 못함
- 즉시 `send-d1-bulk-rest.js`로 보충 발송 (skip set으로 중복 방지)
- **재발 방지**: 모든 Supabase 조회는 `fetchAll()` 페이지네이션 헬퍼 사용 (`scripts/check_recent_sharing.js` 패턴)

### Issue 2 — T1 버튼 URL prefix 충돌
- T1 템플릿은 버튼이 `https://www.selfishclub.xyz/sharing/#{url}` 도메인+경로 하드코딩
- `new.selfishclub.xyz/sharing/aaa?...` 통째 넘기니 `www.../sharing/new.selfishclub.../sharing/...` 이중 prefix
- **재발 방지**: T1 사용 시 `#{url}`에 **slug+UTM만** (`aaa?utm_...`)

### Issue 3 — 신청자 발송 직후 1명 추가 신청 (1,328 → 1,329)
- D-1 발송 시점 신청자 unique = 1,328명
- 미신청자 발송 시점 신청자 unique = 1,329명 (1명 추가 신청)
- 그 1명: D-1 못 받음 + 미신청자 발송에서도 신청자로 분류되어 제외 → **알림톡 미수신 1건**
- **개선 아이디어**: 발송 직전 신청자 phone snapshot을 staticData에 박고, 미신청자 발송에서는 그 snapshot 외 신규 신청자에도 별도 알림 (또는 그냥 누락 허용 — 경계 케이스)

- 미신청자 발송 직후 신규 신청자 빠르게 들어옴 (사용자 발화 기준 "많이 들어왔다")
