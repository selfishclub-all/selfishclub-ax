# outputs/copy-pool — 알림톡 카피 풀 아카이브

## 목적
①② 알림톡 `#{프로그램설명}` 변수용 **카피 풀**(공유회별 5안)을 박제해서:
1. **겹침 방지** — 다음 공유회 copywriter가 이전 풀 참조해서 동일 후크 태그 연속 사용 피함
2. **선택 이력** — 에밀리/주희가 최종 고른 번호 기록 → 피드백 루프
3. **히스토리** — 어떤 후크가 잘 먹혔는지 회고 자료

## 파일 네이밍
```
iid_{N}.md
```
- N = Supabase `item.iid` (예: `iid_79.md`)

## 스키마 (공유회 1건 = 1파일)

```markdown
---
iid: 79
slug: aaa
title: AAA 깃허브 자동화 공유회
event_date: 2026-04-28
created_at: 2026-04-20
generator: copywriter@v2026-04-20
---
```

## 후크 태그 5종 (copywriter 분산 룰)
| 태그 | 각도 | 용례 |
|------|------|------|
| 결핍 | "이거 모르면 손해" | 페인포인트 자극 |
| 가치 | "이거 배우면 뭐 됨" | 혜택/산출물 중심 |
| 사회증거 | "누가/이미 N명이" | 전문가·참여자 언급 |
| 호기심 | "진짜 돼요?" | 의심→해소 구조 |
| 희소성 | "여기서만/한정" | 독점성 강조 |

## 검증
copywriter 생성 후 자동 린트:
```bash
python3 scripts/lint-copy.py outputs/copy-pool/iid_{N}.md
```
**Block 룰**: 글자수 범위 · 후크 태그 허용값 · 금지어/과장 형용사
**Warn 룰**: 표기vs실제 글자수 · 태그 중복(3회+) · 5안 미만

## 규칙
- **동일 후크 태그 연속 3회 금지** (최근 3개 공유회 기준)
- **이전 풀에서 본문 표현 재사용 금지** (완전 동일한 문장)
- 생성 시 최근 3~5건 `iid_*.md` 먼저 읽을 것

## 프로세스
```
1. copywriter 에이전트 생성 → outputs/copy-pool/iid_{N}.md 박제
2. lint-copy.py 자동 검증 (pre-commit hook에서도 실행)
3. dispatcher가 Slack #crm-카피에 포스트
4. 에밀리/주희가 발송 시 번호 선택
5. 선택 이력을 파일 하단에 추가
6. 발송 후 반응 메모 (성과 데이터 붙으면 자동화)
```
