알림톡 = 카톡으로 받는 정보성 메시지 (신청자/멤버십 회원 대상). 노란 배경 그거 맞아요.
알림톡 9종 — 10일짜리 타임라인
D-5 오픈알림 → D-3 오픈리마인드 → D-1 리마인드 → D-day 할인쿠폰 → 당일 리마인드 → 입장링크 → 시작알림 → 혜택안내 → D+3 VOD
매번 대상자도, 카피도, 시점도 다릅니다. 예: D-5는 "오픈알림 받고 안 신청한 사람", D-1은 "신청자만"

08
자동화에 쓴 건 3개 뿐
한 화면에 그리면 이래요.
Supabase DB → n8n WORKFLOW → Slack APPROVAL → SOLAPI 발송
데이터는 Supabase에, 흐름은 n8n이 돌리고, 결정은 Slack에서 사람이 한 번 더.
DB Supabase 회원·신청자·공유회 정보가 다 들어있는 데이터베이스. SQL 손으로 짜던 자리를 대신해요.
WORKFLOW n8n 서버에서 24시간 도는 자동화 도구. 매일 정해진 시간에 알아서 다음 발송을 챙겨줘요.
APPROVAL Slack 발송 직전, 사람이 한 번 더 검수하는 관문. 익숙한 메신저 안에서 끝나요.
