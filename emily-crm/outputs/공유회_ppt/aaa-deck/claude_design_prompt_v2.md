# 🎯 Claude Design 프롬프트

```
셀피쉬클럽 AAA팀 공유회 발표 덱을 다시 만들어줘.
어제 준 v3·v4 초안은 구조 흐름만 참고하고, 비주얼·카피는 아래 피드백 반영해서 새로 제안해줘.
가능하면 섹션별 2~3안 보여주고 내가 고르게 해줘.

# 발표자·상황
- 에밀리, 셀피쉬클럽 CRM PM
- 이기적멤버십 6,800명 공유회·이벤트 CRM 전 채널 혼자 담당
- 최근 6주간 Claude Code로 알림톡 9종 자동화 직접 만듦
- 셀피쉬클럽 AAA팀(8명) 소속, 이번 공유회 발표자
- 발표 시간: 20~25분, 25장 내외, 오프라인 + 녹화

# 청중
반복 업무에 치여 본질에 집중 못 하는 마케터·PM·운영 담당자.
개발 지식 없음. 추상 개념보다 실제 케이스·숫자·스크린샷에 꽂힘.

# 청중이 가져가게 할 3가지
1. AI랑 같이 기획하는 법
2. 자동화 구조 설계 감
3. 내 규칙을 AI가 읽게 만드는 법 (하네스 감)
→ 한마디로 "AI한테 일 잘 시키는 법"

# 디자인 제약 (어제 피드백 반영 + 확정)
- 컬러: 검정 / 흰색 / 셀피쉬 노란색 #E9ED12 (밝기·채도 조정 자유)
- 포인트 컬러는 슬라이드당 1~2곳만. 과하게 쓰지 말 것
- 폰트: Pretendard
- 비율: 16:9
- 원칙: 한 슬라이드 = 한 포인트. 텍스트 최소, 시각(큰 숫자·다이어그램·스크린샷 자리) 중심
- 톤 레퍼런스: 미니멀 타이포 + 인포그래픽 (스트라이프 웹 / 노션 가이드 톤)
- 카피는 "사람이 말하는 톤" — 내 초안 있어도 AI 말투면 다시 써줘

# 전달해야 할 메시지 흐름

## 01. 인트로
- 제목 후킹 "매일 아침, AI가 알림톡을 들고 옵니다" (더 좋은 카피 있으면 제안)
- 내가 누군지 (본업 CRM PM + AAA 크루 이중 정체성)
- AAA팀 8명 크루 소개 (각자 다른 프로젝트)
- 오늘 순서·타겟·가져갈 3가지

## 02. WHY — 지난한 여정 (공감 구간)
핵심: "공유회 CRM이 무지성이 아니라 촘촘한 일이었다"
- 공유회 1건 열리면 4채널(알림톡·카플친·오픈채팅·인스타) 동시 나감
- 알림톡만 봐도 = 10일 × 9번 × 매 시점 대상자 다름
  (D-5는 오픈알림 안 받은 사람만 / D-1은 신청자만 등)
- 매번 수동 SQL로 DB 발라냈음 → 촘촘한 일
- 하루 3시간 → 하루 5분
- "오늘 아침 제 슬랙엔 이게 와요" — 승인 모달 한 번 누르면 끝

## 03. HOW — STEP 1 자동화 구조
- 삽질 ①: Claude Code는 맥북 꺼지면 멈춤 → 서버 필요 → n8n 도입
- 삽질 ②: 완전 자동 무서움 (오타 1번 = 6,800명 민원) → Slack 승인 관문
- 3 도구 역할: n8n (스케줄) / Supabase (DB) / Slack (승인)
- 전체 플로우 다이어그램 1장

## 04. HOW — STEP 2 하네스 (귀납적 흐름 — 매우 중요)
❗ "공간·기억·학습"을 정답 프레임처럼 제시하면 안 됨.
내 삽질 → 질문 → 발견 순서로 자연스럽게:
1. 규칙 너무 많아서 CLAUDE.md 353줄에 다 박음
2. AI가 자꾸 틀리고 까먹음 + 맥북 2개 쓰는데 다른 맥북 켜니 일주일 전 기억이 마지막
3. "하네스" 개념을 AAA팀에서 언뜻 들음 → 클로드에게 직접 질문
   ("너한테 기억 잘 주려면 뭘 가져야 해? 네가 생각하는 하네스는 뭐야?")
4. 적용해보니 잘 돌아감 → "알고 보니 이게 하네스였더라"
5. 정리된 세 축: 공간 · 기억 · 학습

## 04.5 🆕 메모리 실제 작동 장면 (새 슬라이드 — 꼭 넣어줘)
- 위치: 04(공간·기억·학습) 다음 또는 05(유즈케이스 PRD) 옆

  + 폴더 구조 트리 + 3요소 화살표 연결

## 05. 유즈케이스 3개 (필수)
- ① 알림톡 PRD (규칙을 md 파일로 빼서 AI가 읽게)
- ② 에이전트 8개 역할 분담 (director / copywriter / verifier 등)
- ③ 촘촘한 CRM 자동화 (알림톡 ① 발송 후 → data-collector가 신청자 자동 조회 → dispatcher가 제외 발송)

## 06. IMPACT
- 정량: 하루 3시간→5분, 이틀→1시간, 오타·링크 리스크 검수단계 차단
- 정성: 퇴근 직전 카피 쓰다 지쳤던 내가 퇴근할 수 있게 됨 / 맥북 닫아도 나감

## 07. 팀 (뾰족한 구체 케이스만 — 일반론 금지)
"같이 하면 좋아요" 식 NO. 구체 사건만:
- **비비안**(전체 대시보드 크루)이랑 협업 → 내 CRM이 전체 어디 들어가는지 감 잡음
- **다니**(마케팅 OS) 영향 받아 CLAUDE.md → PRD 구조 차용
- **code-reviewer 에이전트** = AAA 과제로 하네스 서로 공유하다 "검수 필요하다" 깨닫고 만듦
- **흐민**(질문하는 AI) / **오웬**(찜마켓) 보면서 활용 범위 확장
→ 그래서 스폰지클럽: 빨아들이고 내보내고 또 누군가가 빨아들이고
```
