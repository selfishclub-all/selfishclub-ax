# 에밀리 AAA 공유회 — 디자인 툴 프롬프트

두 버전 준비: **Genspark용**, **Claude Design용**. 각각 따로 복붙해서 넣으면 됨.

두 프롬프트 모두 레이아웃·카피 직접 지시 **없음**. AI가 맥락·흐름·피드백 읽고 알아서 제안하게 함.

---

## 공통 배경 (둘 다 머리에 둘 것)

### 나
- 에밀리, 셀피쉬클럽 CRM PM
- 이기적멤버십 6,800명 대상 공유회·이벤트 CRM 전 채널 혼자 담당
- 최근 6주간 Claude Code로 알림톡 자동화 만들었음
- 셀피쉬클럽 AAA팀(8명) 소속, 이번 공유회 발표자

### 발표 현장
- 셀피쉬클럽 이기적멤버십 공유회, 오프라인 + 녹화
- 시간: 20~25분 타이트
- 청중 대부분: 마케터·PM·운영 담당자. 개발 지식 없음
- 같은 팀(AAA) 크루 8명 중 여러 명이 각자 발표 → 내 건 그중 하나

### 청중이 가져가게 하고 싶은 것 (3가지)
1. AI랑 같이 기획하는 법
2. 자동화 구조 설계 감
3. 내 규칙을 AI가 읽게 만드는 법 (하네스 감)

한마디로: **"AI한테 일 잘 시키는 법"**

# 전달해야 할 메시지 흐름 (순서대로)
## 01. 인트로
- 제목 후킹 "매일 아침, AI가 알림톡을 들고 옵니다" (필요하면 더 좋은 카피 제안해도 됨)
- 내가 누군지 (본업 + AAA 크루 이중 정체성)
- AAA팀 8명 크루 소개 (각자 다른 프로젝트)
- 오늘 순서·타겟·가져갈 3가지

## 02. WHY — 지난한 여정 (공감)
- 공유회 1건 열리면 4채널(알림톡·카플친·오픈채팅·인스타)로 다 나간다
- 알림톡 하나만 봐도 = 10일 간격 × 9번 발송 × 매 시점마다 대상자 조건 다름
  (예: D-5는 "오픈알림 이후 안 신청한 사람만", D-1은 "신청자만")
- 이걸 매번 수동 SQL로 DB 뽑았음 → 촘촘한 일이지 무지성이 아님
- 결과: 하루 3시간 → 하루 5분. 슬랙 모달에서 승인만 누르면 나감
- "오늘 아침 제 슬랙엔 이게 와요" 실 장면

## 03. HOW — STEP 1 자동화 구조
- 삽질 ①: Claude Code는 맥북 켜져야 돎 → 퇴근하면 못 나감 → n8n 필요
- 삽질 ②: 완전 자동은 무서웠음 (오타 1번 = 6,800명 민원) → Slack 승인 관문 필요
- 3 도구 역할: n8n (스케줄) / Supabase (DB) / Slack (승인)
- 전체 플로우 다이어그램

## 04. HOW — STEP 2 하네스 (귀납적으로 풀 것 — 중요)
※ "공간·학습·기억 프레임워크"를 정답처럼 제시하지 말 것.
내 삽질에서 귀납적으로 도출된 결과라는 흐름을 유지해야 함.
- 규칙이 너무 많아서 CLAUDE.md 353줄에 다 박았음
- 그런데 AI가 또 까먹고 또 틀림 ("어제 말한 규칙 왜 또 틀려?")
- 맥북 2개 쓰는데, 다른 맥북 켜니 일주일 전 기억이 마지막이더라
- 그러다 "하네스"라는 걸 언뜻 들음 → 클로드한테 직접 질문함
  ("너한테 기억 잘 주려면 뭘 가져야 해? 네가 생각하는 하네스는 뭐야?")
- 내 문제에 맞게 적용해보니 잘 돌아감 → "알고 보니 이게 하네스였더라"
- 정리해보면: 공간(파일 쪼갬) · 기억(상태 저장) · 학습(검수 + 피드백 루프)
- 실제 적용 유즈케이스 3개 필수:
  ① 알림톡 PRD (규칙을 파일로 빼서 얘가 읽게)
  ② 에이전트 8개 (director, copywriter, verifier 등 역할 분담)
  ③ 촘촘한 CRM 자동화 (알림톡 ① 발송 후 → data-collector가 신청자 자동 조회 → dispatcher가 제외 발송)

## 05. IMPACT
- 정량: 하루 3시간→5분, 이틀→1시간, 오타·링크 리스크 검수단계 차단
- 정성: 퇴근 직전 카피 쓰다 지쳤던 내가 퇴근할 수 있게 됨

## 06. 팀 (뾰족한 구체 케이스)
※ "같이 하면 좋아요" 같은 일반론 금지. 구체 케이스만:
- 비비안(전체 대시보드 크루)이랑 협업 → 내 CRM이 어디 들어가는지 감 잡힘
- 다니(마케팅 OS) 영향 받아 CLAUDE.md → PRD 구조 차용
- code-reviewer 에이전트는 AAA 과제로 하네스 서로 공유하다 "검수 필요하다" 깨닫고 만듦
- 흐민(질문하는 AI) / 오웬(찜마켓) 보면서 활용 범위 확장
