# 01 — 알림톡 자동화 PRD

## 개요
- SOLAPI 고정 템플릿 9종 (카카오 검수 승인 완료)
- **n8n 독립 운영** — Claude Code 없이 크론 → Slack 모달 → 에밀리 승인 → SOLAPI 발송
- DB 변수 자동 채움 + 모달에서 카피/링크만 수정
- Claude Code는 CRM 시작 시 카피 레퍼런스를 Slack에 올려놓기만 함

## 발송 플로우
```
[CRM 시작 시] Claude Code:
  → 상세페이지 크롤 → 전체 카피 세트 + UTM 생성
  → Slack #crm-카피에 레퍼런스로 올림
  → 이후 알림톡은 Claude Code 관여 안 함

[매일 10:00 KST] n8n 알림톡 크론 (Asia/Seoul):
  → 오늘 할 알림톡 전부 Slack #crm-알림톡-승인단계에 모달 전송
  → 에밀리가 승인/수정/취소
  → 승인 → Wait(scheduledTime까지 대기) → 발송 직전 DB 조회 → SOLAPI 발송
  → 취소 → staticData에 cancel 플래그 → Wait 끝나면 발송 스킵
  → 취소 후 재승인 가능 (cancel 플래그 초기화)
  → 모달에서 발송시간 변경 가능 (비우면 즉시발송)
  → 승인 안 누르면 안 감 (안전)
  → ④⑦은 자동 발송 (모달 없음, staticData 재사용)
```

### URL 처리 규칙
- 모든 link/url에서 `https://` 자동 제거 (Code: 파싱 + Code: 발송 데이터)
- SOLAPI 템플릿 버튼이 `http://#{url}` 또는 `https://#{url}` 형식이라 값에 프로토콜 포함하면 이중 프로토콜 에러
- 모달 placeholder에 "https:// 없이 입력" 안내

### staticData 저장 흐름
```
③ D-1: 줌링크 입력 → staticData[iid].zoomUrl 저장
  → ④ 당일리마인드: 줌링크 자동 (모달 없음)
  → ⑥ 입장링크: 줌링크 자동 (라이브혜택만 입력)
  → ⑦ 시작알림: 줌링크 자동 (모달 없음)

⑥ 입장링크: 라이브혜택 입력 → staticData[iid].benefits 저장
  → ⑦ 시작알림: #{자료집리스트} 자동 (= ⑥ 라이브혜택 재사용, 모달 없음)
  ⚠️ ⑥ 모달에 "⑦ 시작알림 자료집에도 동일하게 들어갑니다" 안내 표시
```

## n8n 크론 스케줄

**단 1개의 크론**이 매일 10:00 KST에 돌고, `Code: 알림톡 판별` 노드가 오늘 날짜에 해당하는 템플릿을 **전부** 뱉는다. 이후 모든 Code 노드가 `runOnceForEachItem` 모드라 하루에 여러 템플릿이 나가도 동시 처리된다.

| 항목 | 값 |
|------|-----|
| 크론 워크플로우 ID | `R2FCAquAvaDPTkWP` |
| Schedule Trigger 포맷 | **6자리** (`초 분 시 일 월 요일`) |
| Cron Expression | `0 0 21 * * *` |
| n8n 서버 타임존 | **America/New_York (EDT/EST)** ← UTC 아님! |
| EDT 21:00 = KST | 다음날 **10:00** |

### ⚠️ 크론 설정 시 주의사항
1. **n8n 서버가 UTC가 아니라 EDT다.** KST 변환 시 `KST - 13시간` (서머타임 때 `KST - 14시간`)
2. **Schedule Trigger는 6자리 포맷이다.** 초(Second)가 맨 앞. `0 0 21 * * *` ≠ `0 21 * * *`
3. **API로 크론 수정 가능.** PUT + deactivate + activate 순서. 단, 이미 지난 시간으로 설정하면 당일 실행 안 됨 (다음날 실행)
4. **모든 Code 노드 `Run Once for Each Item` 모드 필수.** 기본값(Run Once for All Items)이면 9개 입력 중 1개만 처리됨

### 매일 10:00 KST 발송 판별 조건
알림톡 판별 노드가 SQL + 날짜 체크 후 아래 중 **해당되는 것들만** 한번에 뱉는다.

| # | 알림톡 | 트리거 조건 | scheduledTime (KST) |
|---|-------|-----------|-------------------|
| ① | 오픈알림 | `i_submitdate_first = 오늘` | 15:00 |
| ② | 오픈리마인드 | 오픈~공유회 **5일 이상** → `first + 2일 = 오늘`<br>**4일 이하** → `first + 1일 = 오늘` | 13:00 |
| ③ | D-1 리마인드 | `i_eventdate - 1 = 오늘` | 13:00 |
| ④ | 당일리마인드 | `i_eventdate = 오늘` + `staticData.zoomUrl` 있으면 자동승인 | 14:00 |
| ⑤ | 할인쿠폰 | `i_eventdate = 오늘` (D-day, ④⑥⑦⑧과 같은 날) | 13:00 |

> **발송 구조:** 크론은 "오늘 처리할 템플릿 뱉기" 역할만 한다. 승인 후 n8n Wait 노드가 scheduledTime까지 대기하고, **Wait 해제 직후 DB를 다시 조회**하여 발송 직전 시점의 신청자/미신청자 기준으로 대상자를 확정한다. SOLAPI는 예약발송이 아닌 즉시발송이며, Wait이 타이밍을 제어한다.

### 취소 기능
- 승인 후 Slack 취소 버튼 → `staticData[cancel_{t}_{iid}] = true` 플래그 저장
- Wait 해제 후 `Code: 취소 확인` 노드에서 플래그 체크 → true면 발송 스킵 (빈 배열 리턴)
- 취소 후 재승인 시 `Code: SQL 재생성`에서 cancel 플래그 삭제 → 새 Wait 대기
- 승인/취소/재승인 사이클 정상 동작

### 발송 플로우 상세 (승인 후)
```
승인 클릭 → Code: SQL 재생성 (SQL 생성 + scheduledDate 계산 + cancel 플래그 삭제)
  → Wait: 발송시간 대기 (scheduledTime까지 대기)
  → Code: 취소 확인 (cancel 플래그 체크, 템플릿 데이터 staticData._currentSend에 저장)

  → Code: 발송 데이터 (staticData._currentSend에서 템플릿 정보 읽어 SOLAPI 변수 매핑)
  → Switch: 템플릿 → SOLAPI 노드 → HTTP: 결과 보고
```
