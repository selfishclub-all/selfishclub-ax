# 00 — 공유회 CRM 전체 개요

## 아키텍처
```
알림톡: n8n 독립 (크론 → Slack 모달 → 에밀리 승인 → SOLAPI)
나머지: Claude Code (카피/배너 생성 → 대화로 수정 → 확정 → 예약/전달)

Slack = 미리보기 전달용 (버튼 없음)
알림톡 승인 = n8n Slack 모달 Block Kit 버튼
카플친 = Claude Code → SOLAPI 직접/예약 발송
```

## 역할 분담
| 역할 | 담당 |
|------|------|
| n8n (독립, 맥북 꺼도 됨) | 알림톡 9종 모달 발송 + 아침 타임라인 요약 |
| Claude Code (맥북 켜야 함) | 카피 생성, 배너 생성, 카플친 SOLAPI 예약, 오카방/온드 Slack 전달, 인스타 이미지 |
| 에밀리 수동 | n8n 모달 승인, 오카방 복붙, 온드미디어 세팅 |

## 트리거
```
n8n 공유회_CRM_트리거 (매일 09:00 KST)
  → Supabase: 오늘 할 알림톡 + 새 공유회 감지
  → Slack에 아침 타임라인 요약 전달
  → 새 공유회 있으면: "CRM 시작할까요?" 제안
  → 에밀리: Claude Code에서 "CRM 시작해줘 {URL}"
```

## 전체 A-Z 플로우

### Step 1: "CRM 시작해줘 {URL}" (Claude Code)
- URL → 상세페이지 크롤 (제목, 커리큘럼, 혜택, 가격, 공유자, 일정)
- Supabase 조회 (iid, 모집기간, 멤버십 회원 수)
- 타임라인 생성

### Step 2: 전체 산출물 한번에 생성 → Slack 미리보기
CRM 시작 시 **전체 Phase의 전채널 카피/배너/UTM/bit.ly를 한번에 생성**하여 Slack에 스레드로 전달.
에밀리가 Claude Code 대화에서 수정 → 확정 → 예약/전달.

### Step 3: 이후 알림톡은 n8n 자동
Claude Code 관여 없이 n8n 크론이 각 시간에 모달/자동 발송.

→ 확정: 카플친 SOLAPI 예약 / 오카방·온드 에밀리 수동

## 전 Phase 공통 규칙
- 모든 신청 URL에 UTM 자동 부착 (줌링크/노션/VOD 등은 UTM 불필요)
- 알림톡 UTM은 n8n이 `i_formid_webflow` 기반으로 자동 생성
- 오픈채팅용은 bit.ly 변환 필수 (3방 각각 다른 bit.ly, 카피는 각각 다름)
- 카플친 배너는 Slack에 이미지 미리보기 전달
- 이메일은 추후 (오픈 + 마감만)
