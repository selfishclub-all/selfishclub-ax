# 09. 아티팩트 저장 규칙

> 공유회 CRM 산출물(카피, 배너, 캐러셀, UTM 등)의 저장 경로 + 네이밍 SSOT.
> **모든 에이전트는 이 규칙대로 저장**해야 다음 공유회에서 참고 가능.

## 루트 경로
```
.omc/crm-artifacts/{iid}-{slug}/
```
- `{iid}`: `iid79`, `iid999` 형식 (숫자 앞에 iid 프리픽스)
- `{slug}`: 프로그램 슬러그 (예: `ai-github`, `ai-bizvideo`)
- 예시: `.omc/crm-artifacts/iid79-ai-github/`

## 디렉토리 구조

```
.omc/crm-artifacts/{iid}-{slug}/
├── meta.json                     # 프로그램 메타 (eventdate, 공유회명, URL 등)
├── timeline.md                   # 확정된 CRM 타임라인
├── copy/
│   ├── alimtalk.md               # 알림톡 ①②⑤⑧ 카피 레퍼런스
│   ├── openchat.md               # 오픈채팅 3Phase
│   ├── kakaoplus.md              # 카플친 3Phase (배너 카피 + text)
│   ├── instagram.md              # 인스타 캐러셀 카피 + 캡션
│   └── owned.md                  # 온드미디어 4종 (리틀리/대화방소식/웰컴/소식)
├── banners/
│   ├── kakaoplus-open.png        # 카플친 배너 (800×600)
│   ├── kakaoplus-remind.png
│   ├── kakaoplus-close.png
│   ├── kakaoplus-open.html       # 편집 가능한 HTML 소스
│   ├── kakaoplus-remind.html
│   └── kakaoplus-close.html
├── carousel/
│   ├── slide-1.png               # 인스타 캐러셀 (1080×1080)
│   ├── slide-2.png
│   ├── slide-3.png
│   ├── slide-4.png
│   ├── slide-5.png
│   └── source.jsx                # React 소스
├── gemini/
│   ├── bg-open.png               # Gemini 생성 배경 (텍스트 없음)
│   ├── bg-remind.png
│   └── bg-close.png
├── utm.md                        # UTM 21개 + bit.ly 매핑
└── slack-log.md                  # Slack 전달 이력 (Phase별 타임스탬프)
```

## 네이밍 규칙

### 파일명
- 소문자 + 하이픈 (`kakaoplus-open.png`)
- Phase 접미사: `-open` / `-remind` / `-close`
- 버전 금지: 최종본만 유지 (이전 버전은 git 히스토리로)

### 슬러그 (slug)
- URL에서 추출 (예: `selfishclub.xyz/sharing/ai-github` → `ai-github`)
- 한글 금지, 영문 소문자+하이픈

## 저장 시점

| 산출물 | 생성 에이전트 | 저장 시점 |
|--------|-------------|---------|
| `meta.json` | data-collector | CRM 시작 직후 |
| `timeline.md` | team-lead | 타임라인 확정 후 |
| `copy/*.md` | copywriter | 각 채널 카피 생성 완료 시 |
| `banners/*.png` + `.html` | media-ops | 배너 렌더링 완료 시 |
| `carousel/*.png` + `source.jsx` | media-ops | 캐러셀 렌더링 완료 시 |
| `gemini/*.png` | media-ops | Gemini 이미지 생성 직후 |
| `utm.md` | media-ops | UTM 생성 + Google Sheet 기록 후 |
| `slack-log.md` | dispatcher | Slack 전달 건별 append |

## 과거 공유회 참고

copywriter/media-ops는 새 공유회 작업 시작 전 **최근 3건의 아티팩트를 먼저 읽고** 톤/구조를 참고한다:

```
.omc/crm-artifacts/ 하위 디렉토리 최신순 3개 → copy/*.md 읽기
```

## Git 규칙
- `.omc/` 전체가 이미 `.gitignore`에 있음 → 아티팩트는 **로컬 캐시**로만 존재
- 과거 공유회 참고는 로컬 파일시스템 기반 (git 히스토리 아님)
- 머신 이전 시에는 `.omc/crm-artifacts/` 수동 백업 필요
- 필요 시 압축 후 외부 저장소(Google Drive/Notion) 보관 고려

## 삭제/정리 규칙
- 공유회 종료 후 90일까지 보존
- 사후분석 완료 후 핵심 아티팩트(카피/UTM/배너)만 압축 보관 고려

## 마이그레이션 (기존 `outputs/`)
- 현재 `outputs/` 플랫 구조 → `.omc/crm-artifacts/iid79-ai-github/`로 수동 이관 1회 필요
- 이관 후 `outputs/`는 임시 작업 공간으로만 사용 (최종본은 크롤 경로에 저장)
