# 02 — 카플친 (카카오 플러스친구) PRD

## 개요
- 카카오 플러스친구 **와이드형 이미지 메시지** (친구톡)
- Phase별 다른 디자인+카피 (오픈/리마인드/마감/혜택)
- SOLAPI 친구톡 API로 발송
- n8n `카플친_친구톡_발송` 워크플로우 (Webhook → 타겟추출 → Slack 승인 → SOLAPI)

## 발송 스케줄
| Phase | 배너 | 대상 |
|-------|------|------|
| Phase 2 (오픈) | 오픈 배너 | 멤버십 - 신청자 - 수신거부 |
| Phase 3 (리마인드) | 리마인드 배너 | 동일 |
| Phase 4 (D-1) | 마감 배너 | 동일 |
| Phase 6 (Post) | 혜택 배너 | 동일 (프로그램명 변수 교체) |

## 타겟 쿼리
```sql
SELECT m.u_name, m.u_phone FROM member m
WHERE m.u_membership_tf = true
  AND m.u_marketing_agree_tf = true
  AND m.u_phone NOT IN (
    SELECT p.u_phone FROM purchase p
    WHERE p.iid = '{iid}'
      AND (p.p_cancel_amount IS NULL OR p.p_cancel_amount = 0)
  );
```

## 배너 디자인 규칙
- 사이즈: **800×600px** (친구톡 와이드 이미지형 권장)
- 구성: SELFISH CLUB 뱃지 + 서브카피 + 메인카피 + 하단바(날짜+CTA)
- 레이아웃: **풀 이미지 배경 + 오버레이 + 텍스트**
- 글씨 크게 (모바일 축소 시 가독성 — 메인카피 52px+)
- 3개 레이아웃 전부 다르게:
  - **오픈**: 풀 이미지 배경 + 다크 오버레이 + 센터 카피
  - **리마인드**: 풀 이미지 + 하단 그라디언트 + 좌정렬 카피
  - **마감**: 다크 배경 + 레드 글로우 + 긴박 타이포그래피 (이미지 없이)
  - **혜택**: 고정 템플릿 (프로그램명만 교체)

## 이미지 생성 파이프라인
```
1. Gemini API (gemini-2.5-flash-image): 공유회 주제 맞는 배경 이미지 3개 생성
   → 텍스트 없이 비주얼만 (한글 깨짐 방지)
   → 프롬프트에 주제 반영
2. Claude: HTML 조합 (카피 + 이미지 + 레이아웃) → 800×600
3. Puppeteer: HTML → PNG 변환
4. SOLAPI: 이미지 업로드 → 친구톡 와이드형 발송
```

## SOLAPI 발송 구조
- n8n 워크플로우: `카플친_친구톡_발송`
- 트리거: Webhook
- 플로우: Webhook → 타겟추출(Supabase) → Slack 승인 요청 → 승인 시 SOLAPI 발송
- 메시지 구성:
  - image: 업로드된 이미지 URL
  - text: 상세 카피
  - button: CTA 텍스트 + UTM URL

## 카피 규칙
- 오픈: 프로그램 소개 중심, 호기심 유발
- 리마인드: 마감 임박 강조, FOMO
- 마감: 긴급성, 마지막 기회
- 혜택: 참여 혜택 강조 (고정 템플릿)

## 스크립트 참조
- `scripts/solapi-friendtalk.js` — 이미지 업로드 + 친구톡 발송
- `scripts/html-to-png.js` — HTML → PNG 변환
- `scripts/solapi-test.js` — SOLAPI API 테스트
