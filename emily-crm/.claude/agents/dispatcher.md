## 자동 발송 (API)

### 알림톡 (n8n → SOLAPI)
- 워크플로우: `공유회_알림톡_크론` v2 (Published)
- **알림톡은 SOLAPI 고정 템플릿 — 카피 생성 불필요, 변수 매핑만 필요**
- n8n 크론이 자동 감지 → Slack에 미리보기+승인 버튼 → 에밀리 승인 → 발송

#### 크론 스케줄 (단일 크론 구조)

```

### 🚨 n8n 워크플로우 운영 철칙 (실수 방지)

#### 1. Schedule Trigger
- **6자리 크론 포맷 필수**: `초 분 시 일 월 요일` (예: `0 0 21 * * *`)
- **서버 타임존 = America/New_York (EDT/EST)**, UTC 아님
  - KST → EDT: `KST - 13` (서머타임 `KST - 14`)
  - KST 10:00 = EDT 21:00 → `0 0 21 * * *`
- **크론 시간 변경은 n8n UI에서 노드 열고 Publish** (API 업데이트는 간헐적으로 스케줄러 재등록 안 됨)
- 수동 실행 (`Execute workflow from Schedule Trigger`) 후 스케줄이 재등록되는 경향 있음

#### 2. 모든 Code 노드 = `Run Once for Each Item` 모드
- 기본값(`Run Once for All Items`)이면 9개 입력 중 1개만 처리돼서 하루에 1개 템플릿만 나감
- return 포맷: `return { json: {...} };` (배열 X, `[{ json: ... }]` X)
- `$getWorkflowStaticData('global')`은 최상단에서 선언
- `$input.item.json` 사용 (`.first()` 금지 — 항상 첫 아이템만 봄)

#### 3. URL 자동 제거
- 모든 URL 값은 `strip()` 함수로 `http(s)://` 자동 제거
- SOLAPI 템플릿 버튼이 `http://#{url}` 또는 `https://#{url}` 형식이라 프로토콜 포함 시 이중 에러
- 적용 위치: `Code: 파싱` (모달 입력 시), `Code: 발송 데이터` (DB/모달 공통)

#### 4. 수정 모달 규칙
- **모든 템플릿 수정 모달에 발송시간 필드 포함** — `block_id: f_scheduled`
  - `f_time`은 ⑥ 공유회시간 필드가 사용 중 → 중복 시 Slack API 400
  - 기본값/placeholder = 해당 템플릿 `scheduledTime` (14:00, 19:00 등)
  - 비우면 즉시발송
- **모든 버튼 바로 위 경고 워딩 필수**:
  ```
  ⚠️ 승인 시 예약 발송이 확정됩니다. 카피·링크·시간을 반드시 확인 후 승인해주세요.
  ```

#### 5. `Code: 수정 미리보기` cb JSON 필수 필드
재수정 시 값 유지 위해 cb에 반드시 포함:
- `scheduledTime` (발송시간 유지)
- `vodDeadline` (⑨ 열람기한)
- `lh`, `rh`, `cpn`, `ed`, `period`, `link`, `url` (해당 템플릿 변수)

#### 6. SOLAPI 노드 `#{url}` 변수
- ③④⑥⑦⑧⑨ 모두 버튼이 링크 포함 → `variables.#{url}` 매핑 필수
- 누락 시 알림톡 실패 → 문자로 대체 발송됨
- SOLAPI 노드 템플릿 매칭 주의: **case 7 = ⑦ 시작알림, case 8 = ⑧ 혜택안내** (과거 뒤바뀜 버그 있었음)

#### 7. testMode
- `Code: 알림톡 판별` 상단 `testMode:true`일 때 `testTemplates` 배열의 번호만 처리
- 하드코딩 경로: `iid = 'iid_999'` 강제 조회 (프로덕션 SQL 건너뜀 → 실전 검증 불가)
- 프로덕션 검증을 위해서는 **testMode 하드코딩 제거 + iid_999에 실제 member/purchase 세팅** 후 프로덕션 SQL 경로 그대로 타게 해야 함

#### 8. n8n 수동 실행 시 주의
- "Execute workflow from Schedule Trigger" 누르면 Schedule Trigger가 **1개 아이템만 emit**할 수 있음 → 크론 자동 실행과 결과 다름
- 실제 검증은 **매분 크론으로 강제 트리거** (`0 * * * * *`) 후 결과 확인

### 카플친 (Claude Code → SOLAPI 직접/예약)
- n8n 불필요. Claude Code가 배너 생성 → 대화로 확정 → SOLAPI 직접/예약 발송
- 타겟 쿼리:
```sql
WITH applicants AS (
  SELECT DISTINCT u_phone FROM purchase WHERE iid = '{iid}' AND p_cancel_amount = 0
  UNION
  SELECT DISTINCT u_phone FROM event WHERE iid = '{iid}'
)
SELECT m.u_name, m.u_phone FROM member m
WHERE m.u_membership_tf = true AND m.u_marketing_agree_tf = true
  AND m.u_phone IS NOT NULL AND m.u_name NOT ILIKE '%테스트%'
  AND m.u_phone NOT IN (SELECT u_phone FROM applicants WHERE u_phone IS NOT NULL);
```

| 웰컴메시지 | 카카오채널 관리자센터 > 대화방 > 웰컴메시지 | 카피 텍스트 (코드블록) |
| 소식 | 카카오채널 관리자센터 > 소식 > 새 소식 | 카피 텍스트 (코드블록) |

### 오픈채팅 3방 (Phase 2/3/4)
Slack에 방별 카피 전달 → 에밀리가 각 오픈채팅방에 수동 발송:
```
[스레드 1] 🟡 셀피쉬냅다활용
  → 카피 (코드블록, 복사 가능)
[스레드 2] 🟢 셀피쉬냅다실무
  → 카피 (코드블록, 복사 가능)
[스레드 3] 🔵 셀피쉬클럽
  → 카피 (코드블록, 복사 가능)
```

### 인스타 캐러셀 (Phase 3 리마인드)
Slack 승인 후 → **Meta Graph API 자동 게시**:
```
[메인] 📸 인스타 캐러셀 (리마인드)
[스레드] 이미지 1~5 (PNG 파일)
[스레드] 캡션 + 해시태그 (코드블록)
[✅ 승인] → Meta Graph API 캐러셀 자동 게시
```
API 플로우: 이미지 5장 컨테이너 생성 → 캐러셀 컨테이너 → publish
