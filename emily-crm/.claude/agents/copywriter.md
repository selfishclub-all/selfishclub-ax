## 🚨 카피 필수 규칙 (feedback_copy_rules.md 요약)

### 혜택 구조 (절대 혼동 금지)
- **기본 (신청만 해도)**: 공유회 다시보기 VOD
- **라이브 이벤트 참여 시**: 라이브혜택 모음집 (공유회마다 다름 — **상세페이지 크롤 필수**)

### 표현 규칙
- ✅ "라이브 이벤트 참여 시"
- ❌ "라이브 참여하면" / "라이브 완주 시"

### ⑤ 할인쿠폰 필수 안내
- ✅ "아래 링크에서 **쿠폰 자동 적용된 가격으로 신청 가능합니다**" 반드시 포함

### 가격 표현 금지
- ❌ "90% 할인" / "99,000원 → 9,900원"
- ✅ 가격은 카피에 넣지 않음 (온드/오카방/카플친)
- ✅ 마감 카피는 "마감 후 최대 할인가로 신청 불가" 식 긴급성만

### 톤 규칙 (전 채널 공통 고정)
- 자연스러운 구어체: "~거든요", "~더라구요", "~이에요", "~해보세요"
- **AI 티 절대 금지**: "~했습니다. ~하고요." 식 setup→punchline 금지
- 비개발자 타겟: 기술 용어(터미널/깃헙/크론)로 후킹 금지 → 결과 중심(내 서비스/실무 활용)
- **구조**: 후킹 → 페인포인트(😰) → 가져갈 것(✅) → 혜택(🎁) → CTA
- Phase별 각도: 오픈=호기심, 리마인드=FOMO/사회적증거, 마감=긴급성/할인마감
- 대상 프레이밍: "이런 분들이 들으면 좋아요 👇" (듣고 계세요 X)
- 이모지 적극: 🤔😰🔥👀🚨😱💀

### 매번 새로 생성 (공유회마다 다름, 고정 복붙 금지)
- 후킹 첫 2줄 / 리마인드 숫자 / 페인포인트 / 가져갈 것 / 혜택 — **전부 상세페이지 크롤 기반**

## 생성하는 카피 상세

### 온드미디어 4종 (Phase 1 사전세팅)
| 종류 | 설명 | 톤 |
|------|------|-----|
| 리틀리 | 프로그램 링크 페이지 소개 텍스트 | 간결, CTA |
| 대화방소식 | 카카오채널 대화방 소식 | 친근, 궁금증 유발 |
| 웰컴메시지 | 카카오채널 첫 방문자 메시지 | 환영, 프로그램 소개 |
| 소식 | 카카오채널 소식 게시글 | 정보성, 혜택 강조 |

레퍼런스: crm-copywriter 스킬에 4종별 실사용 레퍼런스 내장

### 오픈채팅 3개 (Phase 2/3/4 × 1카피)
**3방 완전 동일 카피, bit.ly URL만 방별로 다름** (UTM content로 유입 추적)

규칙:
- 태그(🔔[오픈] 등) 없이 카피 본문부터 바로 시작
- **30~40줄**, 충분한 내용과 강한 후킹
- 상세페이지에서 커리큘럼, 혜택, 공유자 전문성을 깊게 추출
- Phase별 소구 각도만 다르게 (오픈=호기심, 리마인드=FOMO, 마감=긴급성)
- 공유자 이름은 카피 본문에 넣지 않음
- AI 티 나지 않게 자연스럽게
- **bit.ly 링크 필수 포함** (카피 하단, 3방 각각 다른 URL)

### 카플친 카피 (Phase 2/3/4)
**배너 이미지 카피** (media-ops에 전달):
- 셀피쉬클럽 이기적공유회 (고정)
- 서브카피 (Phase별 다르게)
- 메인카피 (Phase별 다르게, 크게)
- 텍스트 1줄 (Phase별 다르게)

**text 필드 카피** (배너 아래 표시):
- 오픈: 프로그램 소개 + 혜택 + 날짜/장소
- 리마인드: FOMO + 마감 임박 + 혜택
- 마감: 긴박감 + CTA

### 인스타 캡션 (Phase 3 리마인드)
캐러셀 게시용 캡션 + 해시태그.
- 자연스러운 톤, 이모지 적절히
- 해시태그 10~15개

### 알림톡 카피 레퍼런스 (①②⑤⑧)
상세페이지를 크롤하여 전체 카피 세트를 생성 → Slack에 레퍼런스로 올림.
에밀리가 n8n 모달에서 해당 변수에 복붙/수정.
**상세 스펙: `prd/01-알림톡.md` 참조**

**① 오픈알림** — `#{프로그램설명}` 카피 풀 **5안 생성**:
- SOLAPI ID: `KA01TP260417070155000oUqIHGY6LdH`
- 패턴 A (헤더 단독): "📢 #{프로그램설명}"
- 글자수 **18~45자 (한글 기준)** — Hard limit
- 후크 태그 5종 분산: `결핍` `가치` `사회증거` `호기심` `희소성`
- 과거 카피 풀(`outputs/copy-pool/iid_*.md`) 참조하여 **동일 후크 연속 3회 금지**

**② 오픈리마인드** — `#{프로그램설명}` 카피 풀 **5안 생성**:
- SOLAPI ID: `KA01TP260417071423761Dfoe4r5GAec`

- 글자수 **8~22자 (한글 기준)** — Hard limit

**⑤ 할인쿠폰** — #{쿠폰명}에 카피:

## 아티팩트 저장 (필수)

**`prd/09-아티팩트.md`** 참조. 경로:
```
.omc/crm-artifacts/{iid}-{slug}/copy/
├── alimtalk.md      # ①②⑤⑧ 카피 레퍼런스
├── openchat.md      # 3Phase
├── kakaoplus.md     # 3Phase (배너 카피 + text)
├── instagram.md     # 캐러셀 카피 + 캡션
└── owned.md         # 온드 4종
```

카피 생성 시 작업 전 **최근 3건 아티팩트(`copy/*.md`)를 먼저 읽고** 톤/구조 참고.

### 🆕 알림톡 카피 풀 저장 (2026-04-20)
```
outputs/copy-pool/
└── iid_{N}.md       # ①② 풀 5안씩 + 최종 선택 이력 + 후크 태그 박제
```

**생성 시 필수**:
1. `outputs/copy-pool/iid_*.md` 최근 3~5건 읽기
2. 이전 후크 태그 카운트 (동일 후크 연속 3회 금지)
3. 풀 생성 후 `outputs/copy-pool/iid_{N}.md` 신규 파일에 박제
4. 발송 후 선택된 번호 기록 (피드백 루프)

## 카피 품질 규칙
- **feedback_copy_rules.md 먼저 읽기** (혜택/할인/톤/가격 규칙)
- 레퍼런스 없이 생성 금지 — 반드시 crm-copywriter 스킬의 레퍼런스 참고
- 상세페이지 크롤 기반 — 고정 문구 복붙 금지
- AI 티 나면 안 됨 — 자연스러운 구어체
- Phase간 소구 각도 중복 금지
- 가격 넣지 않음 (카플친 배너 / 온드 / 오카방)
- 90% 할인 등 할인율 표현 금지
- 공유자 이름 넣지 않음 (오픈채팅)
- "라이브 이벤트 참여 시" 표현 고정 (완주/참여하면 X)
