# timeline-planner — 타임라인 수립 에이전트

## 참조 PRD
- `prd/00-overview.md` — Phase 구조 + 전채널 액션

## 역할
data-collector가 수집한 프로그램 정보를 기반으로 **전 채널 × 전 Phase 액션 일정표**를 자동 계산한다.
알림톡 9종 + 카플친 4종 + 오카방 9개 + 온드 4종 + 인스타 1회를 **날짜별로 정리**하여 Slack에 전달.

## 입력
- data-collector의 구조화된 프로그램 데이터 (eventdate, submit_first, submit_last 등)

## 출력
Phase별 날짜 + 시간 + 채널 + 발송방식이 포함된 타임라인

## 타임라인 계산 로직

### Phase 배정
```
Phase 1 사전세팅: submit_first - 2일 (오픈 전)
Phase 2 오픈: submit_first (모집 시작일)
Phase 3 리마인드: submit_first + 2일 (오픈리마인드는 D+2 자동) 또는 오픈~마감 중간
Phase 4 D-1: eventdate - 1일
Phase 5 D-day: eventdate
Phase 6 Post: eventdate + 1일~
```

### Phase별 채널 + 시간
| Phase | 채널 | 시간 | 발송방식 |
|-------|------|------|---------|
| 1 사전세팅 | 온드미디어 4종 (리틀리/대화방소식/웰컴메시지/소식) | 오픈 전 아무때나 | Slack→수동 |
| 2 오픈 | 알림톡①(멤버십전체) + 카플친 + 오카방3 + 이메일 | 오픈 당일 | 자동+수동 |
| 3 리마인드 | 알림톡②(미신청자,10:00) + 카플친 + 오카방3 + 인스타캐러셀 | D+2 자동 | 자동+수동 |
| 4 D-1 | 알림톡③(신청자,13:00) + 카플친 + 오카방3 + 이메일 | 전날 | 자동+수동 |
| 5 D-day 13:00 | 알림톡⑤ 할인쿠폰(미신청자) | 13:00 | n8n 모달(쿠폰명+종료일) |
| 5 D-day 14:00 | 알림톡④ 당일리마인드(신청자) | 14:00 | ❌ 자동(줌링크③) |
| 5 D-day 19:00 | 알림톡⑥ 입장링크(신청자) | 19:00 | n8n 모달(라이브혜택) |
| 5 D-day 20:00 | 알림톡⑦ 시작알림(신청자) | 20:00 | ❌ 자동(줌링크③+자료집⑥) |
| 5 D-day 22:00 | 알림톡⑧ 혜택안내(신청자) | 22:00 | n8n 모달(카피+노션, 오전입력) |
| 6 D+3 18:00 | 알림톡⑨ VOD발송(신청자) | D+3 | n8n 모달(VOD링크, 오전입력) |
| 6 Post | 카플친혜택 + 이메일VOD | D+1~ | Claude Code + 추후 |

### 리마인드 타이밍
- **오픈+2일 고정** (1회만)

### VOD 발송
- **D+3, 18:00** — 오전에 모달로 VOD링크 입력

## 전 Phase 공통 규칙
- 모든 신청 URL에 UTM 자동 부착 (줌링크/노션/VOD는 UTM 불필요)
- 오픈채팅용은 bit.ly 변환 필수
- 이메일은 오픈 + 마감만 (리마인드 없음)
