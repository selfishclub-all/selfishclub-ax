# selfishclub-detail-page

셀피쉬클럽 프로그램 상세 페이지 콘텐츠 기획 및 HTML 생성 스킬. 프로젝트에 업로드된 01-copywriting-guide.md와 02-design-guide.md를 반드시 읽은 후, 단계별 인터뷰 → 섹션별 기획 대화 → HTML Artifact 생성 → 이미지 추천까지 전체 과정을 수행한다. 사용자가 '상세 페이지', '상세페이지', '공유회 페이지', '프로그램 페이지', '랜딩 페이지', '페이지 만들어', '상세 만들어' 등을 언급하면 반드시 이 스킬을 사용할 것.

**최종 산출물**: HTML Artifact — 사용자가 어드민에 복사-붙여넣기하여 상세 페이지를 완성할 수 있도록 한다.

---

## 시작 전 필수

스킬이 실행되면 **가장 먼저** 프로젝트에 업로드된 아래 파일 2개를 읽는다:
- `01-copywriting-guide.md` — 카피 톤, 표현 규칙, 후기 인사이트
- `02-design-guide.md` — 색상, 폰트, HTML 코드 샘플

이 파일을 읽지 않은 상태에서 어떤 콘텐츠도 생성하지 않는다.

---

## 워크플로우 개요

```
[인터뷰: 한 문항씩] → [섹션별 기획 대화] → [카피 종합 확인] → [HTML Artifact] → [수정] → [최종 전달]
```

---

## Phase A: 인터뷰 (한 문항씩 진행)

**반드시 한 번에 하나의 질문만 한다.** 여러 질문을 한꺼번에 던지지 않는다.
각 질문에는 선택지를 제공하여 사용자가 골라서 답할 수 있게 한다.
사용자가 답하면 다음 질문으로 넘어간다.

### Q1. 프로그램 유형

```
어떤 프로그램인가요?

□ 이기적공유회
□ 이기적챌린지
□ 워크숍
□ 기타 (직접 입력)
```

### Q2. 기본 정보

```
프로그램의 기본 정보를 알려주세요.

- 제목:
- 날짜/시간:
- 가격: □ 무료  □ 유료 (금액:      원)
```

### Q3. 공유자 & 세션

```
공유자와 세션 정보를 알려주세요.

- 공유자1 이름:
- 공유자1 세션 주제:
- 공유자2 이름: (없으면 '없음')
- 공유자2 세션 주제:
(공유자가 더 있으면 추가해주세요)
```

### Q4. 회의 스크립트 (건너뛰기 가능)

```
기획 회의 녹음 스크립트나 메모가 있나요?
있으면 아래에 붙여넣어주세요. 핵심 내용을 추출해서 카피에 반영할게요.

□ 있어요 → 아래에 붙여넣기
□ 없어요, 건너뛸게요
```

사용자가 스크립트를 붙여넣으면:
- 핵심 주제, 공유자별 다루는 내용, 인상적인 표현, 구체적 수치를 추출한다
- "스크립트에서 이런 내용을 추출했어요. 맞나요?" 확인을 받는다

### Q5. 큰 주제

```
이번 프로그램의 핵심을 한 줄로 요약하면?

예시:
□ "비개발자도 앱을 만들 수 있다"
□ "AI 영상으로 실제 매출을 만든 과정"
□ 직접 입력:
```

(스크립트가 있었다면 스크립트에서 추출한 내용을 선택지로 제안한다)

### Q6. 참여자가 얻어갈 한 가지

```
참여자가 이걸 듣고 "아, 나도 이건 해봐야겠다"라고 느꼈으면 하는 한 가지는?

예시:
□ "나도 코드 없이 뭔가 만들 수 있겠다"
□ "이 워크플로우 바로 따라 해봐야지"
□ 직접 입력:
```

### Q7. 임팩트 사례 (건너뛰기 가능)

```
이번 공유회에서 가장 임팩트 있는 장면이나 사례가 있나요?
(예: "5시간 만에 서비스 런칭", "비용 90% 절감" 등)

□ 있어요:
□ 건너뛸게요
```

### Q8. 지향점

```
이번 프로그램에서 특별히 강조하고 싶은 방향이나 포인트가 있나요?

예시:
□ 비개발자도 할 수 있다는 것 강조
□ 실전 워크플로우 전체 공개 강조
□ 삽질/시행착오 과정의 진정성 강조
□ 직접 입력:
□ 특별히 없어요
```

### Q9. 조심할 점

```
피해야 할 표현이나 방향이 있나요?

예시:
□ 특정 툴을 광고하는 느낌 금지
□ 너무 쉽다는 표현 금지 (오히려 삽질 강조)
□ 경쟁사/타 커뮤니티 언급 금지
□ 직접 입력:
□ 특별히 없어요
```

### Q10. 타겟

```
어떤 사람들에게 와달라고 하고 싶나요?

예시:
□ AI 툴은 써봤지만 실무 적용을 못 하는 사람
□ 코딩을 전혀 모르는 마케터/기획자
□ 혼자 공부하다 막힌 사람
□ 직접 입력:
```

### 인터뷰 규칙
- **한 번에 한 문항만** — 절대 2개 이상 동시에 묻지 않는다
- **선택지 필수** — 모든 질문에 골라서 답할 수 있는 선택지를 제공한다
- **건너뛰기 존중** — 사용자가 "건너뛸게", "없어", "넘어가"라고 하면 즉시 다음으로
- **"바로 만들어줘" 존중** — 남은 질문을 모두 건너뛰고 Phase B로 즉시 이동
- Q1~Q3은 필수, Q4~Q10은 건너뛰기 가능

---

## Phase B: 섹션별 기획 대화

인터뷰 답변을 바탕으로, **섹션을 하나씩** 기획한다.
각 섹션마다 내가 먼저 초안을 제안하고, 사용자가 확인/수정한다.
"알아서 해줘"라고 하면 내가 채우고 다음으로 간다.

### 섹션 순서

**1. Hook** — 2~3개 후보를 제안, 사용자가 선택
**2. Summary** — 초안 1개를 보여주고 피드백
**3. Curriculum** — 세션별 제목 + 항목을 제안, 빠진 내용 확인
**4. Speakers** — 공유자별 소개 + Before→After를 함께 만듦 (건너뛰기 가능)
**5. Target** — 4개 후보를 제안, 추가/수정 확인
**6. Benefits** — 혜택 목록 제안 (건너뛰기 가능)
**7. 이미지/GIF 계획** — 최소 5개 슬롯 제안 (건너뛰기 가능)

### 섹션별 대화 규칙
- 한 턴에 한 섹션씩
- 항상 초안을 먼저 제안 — 빈 칸을 주지 않는다
- 선택지 제공 — 2~3개 후보를 주거나 A/B/C로 고르게 한다
- 같은 섹션에서 2번 이상 되묻지 않는다
- "바로 만들어줘"라고 하면 남은 섹션을 모두 내가 채우고 Phase C로 점프

---

## Phase C: 카피 초안 종합 확인

모든 섹션이 채워지면 전체를 한 번에 정리해서 보여준다:

```
📄 전체 카피 초안

🎯 Hook: "..."
📝 Summary: ...
📋 Curriculum: ...
👤 Speakers: ...
🎯 Target: ...
🎁 Benefits: ...
📷 이미지 계획: ...
```

"이대로 HTML을 만들까요? 마지막으로 수정할 부분 있으면 알려주세요."

**이 확인을 건너뛰지 않는다.**

---

## Phase D: HTML Artifact 생성

확인되면, **02-design-guide.md의 HTML 코드 샘플을 정확히 따라** 전체 상세 페이지를 Artifact로 생성한다.

### 생성 규칙
- Artifact(디자인 미리보기)로 생성
- 인라인 스타일만 사용 — 외부 CSS 없이 단독 동작
- 폰트: `font-family: 'Pretendard', -apple-system, sans-serif`
- 이미지/GIF 슬롯은 점선 플레이스홀더 + 구체적 추천 내용
- **FAQ, 공통 블록, 멤버십 안내는 포함하지 않음**

---

## Phase E: 수정

사용자가 수정을 요청하면 해당 부분만 바꿔서 Artifact 업데이트.
수정은 원하는 만큼 반복 가능.

---

## Phase F: 최종 전달

최종 확인 후:

1. "이 HTML을 어드민에 붙여넣으면 상세 페이지가 완성됩니다." 안내
2. Artifact 복사 버튼 안내
3. 준비할 이미지/GIF 목록을 별도로 정리:

```
📷 준비할 이미지/GIF 목록:
1. [위치] — 구체적 설명
2. [위치] — 구체적 설명
...
```

---

## 절대 규칙

1. **시작 전에 01-copywriting-guide.md, 02-design-guide.md를 반드시 읽는다**
2. **인터뷰는 한 문항씩** — 여러 질문을 한꺼번에 던지지 않는다
3. **Phase C(종합 확인)는 절대 건너뛰지 않는다**
4. **FAQ는 HTML에 포함하지 않는다**
5. **카피라이팅 가이드의 톤을 지킨다** — "혁신적인", "완벽한", "~해드립니다" 금지
6. **이미지/GIF 슬롯은 최소 5개**
7. **Before→After는 구체적 상황으로**
8. **Target은 직군이 아니라 고민으로**
9. **디자인 가이드의 HTML 패턴을 따른다**
