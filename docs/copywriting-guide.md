# 셀피쉬클럽 카피라이팅 가이드

> 이 문서는 AI 초안 생성 시 프롬프트에 자동 포함됩니다.
> 공유회/챌린지 상세페이지의 톤, 구조, 표현 규칙을 정의합니다.

---

## 1. 브랜드 정체성

- **셀피쉬클럽**: AI로 나의 가능성을 확장하는 커뮤니티
- **이기적공유회**: 60%의 지식경험을 100%로 만들어가는 공유자를 위한 공유. 배우는 공유가 아니라 **공유자가 가장 크게 성장하는 공유**
- **핵심 가치**: 진솔함, 실전, 삽질의 가치, 함께하는 성장

---

## 2. 톤 앤 매너

### 원칙
- **진솔하고 날것** — 성공 스토리가 아니라 삽질 과정을 보여준다
- **과장 없는 공감** — "혁신적인", "완벽한" 대신 "부딪쳐봤다", "직접 해봤다"
- **대화하듯 편하게** — 격식체(~합니다)와 반말(~거예요, ~했어요)을 자연스럽게 섞음
- **구체적 숫자** — 막연한 성과 대신 "조회수 185% 상승", "리소스 90% 절감" 같은 구체적 수치

### 선호 표현
| 쓰는 표현 | 안 쓰는 표현 |
|---|---|
| 직접 부딪쳐봤습니다 | 혁신적인 프로그램 |
| 있는 그대로 공유합니다 | 완벽한 솔루션 |
| 우당탕탕 삽질기 | 체계적인 커리큘럼 |
| 싹-다 공개합니다 | 종합적으로 다룹니다 |
| 냅-다 해봤습니다 | 전략적으로 접근합니다 |
| 날 것 그대로 | 정제된 인사이트 |
| ~인데, 어떻게 했을까요? | ~에 대해 알아봅시다 |
| 쌩-날 기록 | 성공 사례 |
| 찐 경험담 | 전문가 강의 |
| 이렇게 달라졌어요 | 성과를 달성했습니다 |

### 문체 패턴
- **질문형 도입**: "공들여 만든 콘텐츠, 사람들이 보질 않아요.. 공감하시나요?"
- **반전 구조**: "처음엔 ~했어요. 결과는 실패. 그래서 방향을 바꿨어요."
- **Before → After**: 구체적 상태 변화를 대비시켜 보여줌
- **이모지 절제**: 핵심 포인트에만 사용 (✔️, 🎯, 🎁, ⚠️), 남발 금지

## 3. 섹션별 작성 규칙

### 3-1. Hook (히어로 문구)
- 한 줄짜리 강렬한 질문 또는 선언
- 독자의 고민을 직접 건드리는 형태
- 예시:
  - "AI, '딸-깍'이 가능할까요?"
  - "누구나 가능은 거짓말!"
  - "공들여 만든 콘텐츠, 사람들이 보질 않아요.."
  - "실제 주문까지 이끈 AI로 만드는 나만의 리뷰 담당자"

### 3-2. Summary (소개 문구)
- 2~4문장 구성: **문제 제기 → 공감 → 이 공유회의 존재 이유**
- 독자가 "아, 이거 내 얘기다" 느끼게
- 예시 패턴:
  ```
  매일 쏟아지는 리뷰에 일일이 답하기 힘든 브랜드 운영자,
  "이걸 다 봐야 하나요?"라는 생각이 들었던 적 있다면,
  이번 공유회가 도움이 될 거예요.
  ```

### 3-3. Curriculum (커리큘럼/아젠다)
- PART 번호 + 제목 + 공유자명
- 각 PART에 ✔️ 체크리스트 형태의 세부 항목 3~5개
- 제목은 **질문형** 또는 **결과 중심**으로
  - "셀피쉬클럽은 고객 데이터를 어떻게 관리할까?"
  - "3초를 바꾸니 일어난 일"
  - "맨땅에서 시작한 AI 해외영업, 0에서 성과까지"

### 3-4. Speakers (공유자 소개)
- 이름 + 역할/타이틀 (1줄)
- 세션 제목 (질문형 또는 액션형)
- 소개 2~3문장: 무엇을 했고, 왜 이 사람이 공유하는지
- **Before → After 카드**: 6주 전/지금 상태를 구체적으로 대비
  ```
  6주 전: 혼자 AI를 학습하고 적용하는 데 한계를 느꼈다.
  지금: 8명이 매주 모여 서로 만든 것을 주고받으며 6주를 채웠다.
  ```

### 3-5. Target (추천 대상)
- 4개 항목, 번호 매김 (01, 02, 03, 04)
- **구체적 상황 묘사** — 직군이 아니라 고민을 적음
  - "AI 툴은 쓰지만 조회수가 안 나오는 크리에이터" (O)
  - "크리에이터" (X)
  - "리뷰 대응마다 브랜드톤이 달라 고민인 브랜딩담당자" (O)
  - "브랜딩 담당자" (X)

### 3-6. Timetable (진행 순서)
- 시간 + 제목 + 간단 설명
- PART 구분 명확하게

### 3-7. Benefits (혜택)
- "신청만 해도" / "라이브 참여시"로 구분
- 구체적 혜택명 명시
  - "공유회 다시보기 VOD" (O)
  - "다양한 혜택 제공" (X)
- 번호 매김 (01, 02, 03)

### 3-8. FAQ
- Q&A 형태, 3~5개
- 자주 반복되는 공통 FAQ는 별도 관리 (아래 공통 블록 참조)

### 3-9. Image Suggestions (이미지/GIF 추천)
- 각 섹션에 어떤 시각 자료가 들어가면 효과적인지 구체적으로 제안
- 예: "공유자 프로필 사진", "Before/After 비교 인포그래픽", "실제 대시보드 스크린샷"

## 4. Before → After 패턴

셀피쉬클럽의 시그니처 표현 방식. 공유자 소개 또는 프로그램 소개에 활용.

### 형식
```
As-is (이전)              →  To-be (이후)
구체적 문제 상황 묘사         구체적 변화 결과 묘사
```

### 예시
| As-is | To-be |
|---|---|
| MAKE 자동화 구독 연 240만원 | n8n 서버 구축 시 0원, 비용 100% 절감 |
| 공유회마다 자동화 구축 1시간 | 워크플로우 1개로 무한 사용, 운영 리소스 90% 절감 |
| "딸깍 자동화"를 믿고 시도했지만 실패 | 나만의 에디터를 만들어 실제 업무에 쓰고 있다 |

| 유형 | 톤 특징 | 가격 | 혜택 구조 |
|---|---|---|---|

| **이기적챌린지** | 직접 해보는 실습, 과제 중심 | 유료 | 실습 템플릿 + 피드백 |

## 7. 절대 하지 않는 것

- "최고의", "완벽한", "혁신적인" 같은 과장 수식어
- "~해드립니다", "~을 제공합니다" 같은 기업 보도자료 톤
- 영어 남발 (필요한 경우에만, 한국어 병기)
- 이모지 과다 사용 (핵심 포인트에만)
- 근거 없는 성과 주장 (숫자가 있으면 구체적으로, 없으면 안 씀)
- "누구나 쉽게" 같은 표현 (오히려 "누구나 가능은 거짓말!"이 셀피쉬 톤)
